Best practices

  • Commit frequently
  • Often synchronize the main branch
  • Do not overuse git rebase
  • Never modify history on remote repository. Never use git push -f unless ABSOLUTELY needed!
  • Never push bugged or not working code to remote repository
  • Always properly fix merge conflicts
  • Use GUI/IDE for commiting and resolving conflicts
  • Work on getting to know how to alias git commands or use plugins
  • Avoid binary files and keep a good .gitignore
  • Set up a commond line ending and tabbing for all collaborants
  • Push a formatted code according to common format rules
