This tutorial is intended to be performed in pairs. You should know how to set up a GitHub repository and add collabolators to it. You should also know how to set up SSH key on GitHub

Optionally you could install Python 2.7 interpreter to run created scripts :)

Config

Console

  1. Start the git console. It is either Git Shell on Windows or the usual terminal on Linux

Git

  1. Configure user data: 
    ~ git config --global user.name "YOUR_USERNAME"
~ git config --global user.email "me@here.com"
  2. Configure merge conflict tool (e.g. kdiff3): 
    ~ git config --global merge.tool kdiff3
~ git config --global mergetool.kdiff3.path 'C:\Program Files (x86)\KDiff3\kdiff3.exe'

GitHub repository

  1. One person creates a GitHub repository
  2. Second person is added as a collabolator to the created repository

`Mayday` script

  1. Download zipped file:
    • Windows
    • Linux
  2. Edit file config.ps1 (Windows) or config.sh (Linux) and add SSH address of created GitHub repository: 
    $github_url = "git@github.com:user/workshop.git"    [WINDOWS]
github_url="git@github.com:user/workshop.git"       [LINUX]
  3. Usage: 
    ~ .\mayday.ps1 N U    [WINDOWS]
~ ./mayday.sh N U     [LINUX]
    Where N means number of state to which the repository should be rewound and U (0 or 1) means if the remote repository should be updated
    If you work in a pair then the second person should run the script with the same parameters (one of you should have U=1)

Creating working directory

  1. Let's run: 
    ~ .\mayday.ps1 0 0    [WINDOWS]
~ ./mayday.sh 0 0     [LINUX]
  2. Go to ./gitlabs/1 for Person1 or ./gitlabs/2 for Person2
