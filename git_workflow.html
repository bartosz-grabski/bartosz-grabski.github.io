Git workflow

Properly implemented workflow may streamline development and keep repository clean. One of the best workflow I came across is the one suggested by Vincent Driessen in his web article

Vincent suggests using of several branches and successfully switching between and merging. Sticking to a model like this maintains a clean and transparent commit history. Also working on new features is really simple, just create a new feature_branch of develop, work on the feature and then merge back to develop. Master branch should contain snapshot of releases. The rest are development branches.
